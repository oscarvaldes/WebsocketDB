To whom it may concern:

After reading the GitHub repo, I hope most of the functionality and application structure makes sense. The goal of this application is to mimic the current functionality of the AG-database (ask Rick to show you if you have not seen it); however, the main difference between this application and the current DB is that this application works with a MySQL database. There are many advantages to a MySQL DB over a traditional Access DB (worth researching).

Getting Started:

Your four most used commands from the server terminal will be (make sure you are in the project directory when running these commands):

  • pm2 status //will give you valuable information about the application ex: if it’s running or if it’s down; sometimes the app may stop running if Rick has done a server restart

  • pm2 start processes.json –env production //to start the application if it is not already running

  • pm2 stop processes.json //stops the application

  • pm2 logs //probably my most used command because it allows you to print stuff to the console server-side and debug
